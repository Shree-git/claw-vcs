# Dependency Policy

This repository uses layered dependency checks:

- `cargo audit` for RustSec advisories.
- `cargo deny check` for advisories, licenses, duplicate versions, and allowed sources.
- GitHub Dependency Review for pull requests.
- Dependabot for security and ecosystem updates.
- SBOM generation for releases.

## cargo-vet

`cargo-vet` metadata lives in `supply-chain/` and is enforced in CI. Run:

```bash
cargo vet
```

Initial metadata starts with exemptions generated by `cargo vet init`; replacing
those exemptions with real audits is part of routine dependency maintenance.

## Update routing

- Security-only updates may be reviewed on an accelerated path when the advisory
  affects a shipped or release-candidate artifact.
- Runtime dependency additions need a reason in the PR body and should update
  SBOM/release notes when user-visible.
- Build-only dependency additions should explain why the existing toolchain is
  insufficient.
- New network-facing dependencies need security review before release.
